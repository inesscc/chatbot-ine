    Paso 3 — Configurar la API¶

    El chatbot usa un servicio externo de inteligencia artificial (OpenRouter) para interpretar tus preguntas. Para esto necesitas una clave de API: una contraseña personalizada que identifica tu cuenta y autoriza el uso del servicio. Si es que quisieran tener su propia cuenta, estos son los pasos a seguir:

    3.1 Crear una cuenta en OpenRouter¶

    1. Ve a openrouter.ai
    2. Haz clic en "Sign In" → "Create account"
    3. Puedes registrarte con tu cuenta de Google o con email

    3.2 Obtener tu clave de API¶

    1. Una vez dentro, haz clic en tu perfil (arriba a la derecha)
    2. Ve a "API Keys"
    3. Haz clic en "Create Key", ponle un nombre (ej: chatbot-magister) y copia la clave

    La clave se verá así: 

    sk-or-v1-abc123def456ghi789...

    Guarda tu clave en un lugar seguro

    Esta clave es como una contraseña. No la compartas públicamente ni la incluyas en archivos que vayas a subir a internet.

    3.3 Agregar créditos¶

    OpenRouter es de pago, pero muy económico para uso académico. Los modelos de este proyecto cuestan fracciones de centavo por consulta.

    1. En OpenRouter, ve a "Credits"
    2. Agrega créditos (se recomienda empezar con USD $5, suficiente para cientos de consultas)

    3.4 Crear el archivo de configuración¶

    Debes crear un archivo llamado .env dentro de la carpeta del proyecto. Este archivo guarda tu clave de API y otras configuraciones.

    ¿Por qué no viene incluido el archivo .env?

    El archivo .env contiene información sensible (claves de API). Por seguridad, nunca se incluye en el código compartido y cada persona debe crear el suyo propio.

    Abre la terminal, navega a la carpeta del proyecto y crea el archivo con el editor nano:

    cd /Users/tu_nombre/proyectos/chatbot/open-webui
nano .env

    Se abrirá un editor de texto en la terminal. Escribe lo siguiente, reemplazando TU_CLAVE_AQUI con tu clave real:

    LLM_SERVICE=openrouter
OPENAI_MODEL=qwen/qwen3-32b
OPENAI_API_KEY=TU_CLAVE_AQUI
OPENAI_API_BASE_URL="https://openrouter.ai/api/v1"
ENABLE_PERSISTENT_CONFIG=True
GLOBAL_LOG_LEVEL=20

    Para guardar y salir: presiona Ctrl + X, luego Y, luego Enter.

    Abre PowerShell, navega a la carpeta del proyecto y abre el Bloc de notas:

    cd C:\Users\tu_nombre\proyectos\chatbot\open-webui
notepad .env

    Si pregunta si quieres crear el archivo, di Sí. Escribe lo siguiente, reemplazando TU_CLAVE_AQUI con tu clave real:

    LLM_SERVICE=openrouter
OPENAI_MODEL=qwen/qwen3-32b
OPENAI_API_KEY=TU_CLAVE_AQUI
OPENAI_API_BASE_URL="https://openrouter.ai/api/v1"
ENABLE_PERSISTENT_CONFIG=True
GLOBAL_LOG_LEVEL=20

    Guarda con Ctrl + S y cierra el Bloc de notas.

    Nombre del archivo

    Asegúrate de que el archivo se llame .env y no .env.txt. Si Windows le agrega .txt automáticamente, renómbralo: clic derecho → Cambiar nombre → elimina el .txt al final.

    3.5 Verificar el archivo¶

    cat .env
    Deberías ver las líneas que escribiste, con tu clave real en OPENAI_API_KEY.

    type .env
    Deberías ver las líneas que escribiste, con tu clave real en OPENAI_API_KEY.

    Si el archivo tiene el contenido correcto, continúa al siguiente paso.
